# ADR.md — Permanent Architecture Decision Record

Write this file alongside your analysis — you have the most context right now.
This file survives the session. Write it as if explaining to an engineer who joins in 6 months.

```markdown
# ADR — [feature name]

## Status
Accepted | Superseded by ADR-XX | Deprecated

## Context
[The problem and why it needed an architectural decision]

## Decision
[What was decided — be specific and unambiguous]

## Rationale
[Why this decision. What alternatives were considered and why rejected.]

## Consequences
**Positive:** [what becomes easier or safer]
**Negative / Trade-offs:** [what becomes harder or more constrained]
**Risks:** [edge cases, performance, security surface]
```
